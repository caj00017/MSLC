You are an aerospace systems engineer, Mars surface-systems analyst, mission logistics modeler, and senior software developer. Build a Mars Surface Logistics Calculator that estimates the landed mass, cargo mission count, setup sequence, power balance, consumables, ISRU production, and resupply cadence required to sustain a configurable number of humans on the Martian surface using Starship-sized cargo missions.

The developer does NOT have access to the original lunar reference spreadsheets. All spreadsheet-derived information needed to recreate the modeling approach is embedded below. Do not ask for the spreadsheets. Use the embedded values as legacy lunar-analog starter assumptions only. Build the Mars calculator so every number can be edited, imported, exported, and replaced by better engineering data later.

The calculator must be auditable, unit-consistent, scenario-driven, and easy for non-programmers to use.

Primary objective:

Create a calculator that answers:

1. How much landed mass is required to establish minimal survival capability for N humans on Mars?

2. How much additional landed mass is required for exploration and science capability?

3. How many Starship-sized cargo missions are required before crew arrival?

4. How many missions are required for each operating mode:

   - Survival

   - Survival + exploration

   - Survival + exploration + science

   - Growth/thrive mode with greenhouse, construction, expanded ISRU, and optional propellant production

5. What mass and cadence are required for resupply?

6. What consumables, spares, and replacement hardware dominate sustainment mass?

7. What is the limiting resource: mass, power, energy storage, water, oxygen, food, ISRU throughput, excavation throughput, crew capacity, or dependency sequencing?

- Mars sol default: 24.65979 hours.

- Default packing efficiency: 0.90.

Core modeling principles:

- Never mix kW and kWh.

- kW = instantaneous power.

- kWh = energy over time.

- Use sols for Mars surface operations but also display Earth days.

- Keep crew-life-support oxygen separate from propellant oxygen.

- Keep return propellant production optional.

- Do not allow crew arrival until readiness checks pass.

- Do not hard-code lunar values as Mars facts.

- Every assumption must be editable in the UI and stored in a scenario file.

EMBEDDED LEGACY LUNAR-ANALOG REFERENCE MODEL

============================================================

The original lunar reference material had two model types:

1. Node-based life-support/ISRU/power model

   It used these nodes:

   - Habitat node: consumes water, oxygen, and power; outputs waste.

   - ISRU node: consumes power and regolith; produces oxygen.

   - Excavation node: consumes power; produces regolith/feedstock.

   - Power generation node: sums habitat, ISRU, excavation, greenhouse, science, and spaceport energy.

   - Greenhouse node: consumes water and power; placeholder for food production.

   - Science node: consumes power.

   - Spaceport node: consumes power and oxygen; placeholder for water production and propellant support.

2. Asset catalog and mission-packing model

   It used:

   - Asset rows with mass, power, duty cycle, quantity, source, and notes.

   - Day power calculation.

   - Night energy calculation.

   - Battery sizing using specific energy.

   - Cable and rover sizing.

   - Total mass rollup.

   - Mission-set packing with mass, power, energy, and dependency checks.

Known issues in the legacy spreadsheets:

- Some values were explicitly marked "PLACE HOLDER VALUES."

- Some cells included notes such as "check," "Ask Jibu about these," and "circular reference here."

- Some power-generation cells produced `#DIV/0!`.

- The calculator must rebuild the logic cleanly and not reproduce spreadsheet errors.

============================================================

CORE FORMULAS

============================================================

1. Time conversion

hours_per_sol = 24.65979

earth_days_per_sol = hours_per_sol / 24

surface_duration_earth_days = surface_duration_sols * earth_days_per_sol

resupply_window_earth_days = resupply_window_months * 30.4375

resupply_window_sols = resupply_window_earth_days / earth_days_per_sol

2. Starship usable payload

usable_payload_kg =

  landed_payload_kg_per_starship

  * packing_efficiency

  * (1 - unallocated_margin_percent)

cargo_missions_required = ceil(total_landed_mass_kg / usable_payload_kg)

3. Gross crew consumables

gross_resource_kg =

  crew_count

  * rate_kg_per_crew_sol

  * duration_sols

4. Oxygen

gross_O2_kg =

  crew_count

  * oxygen_kg_per_crew_sol

  * duration_sols

recovered_O2_kg =

  gross_O2_kg

  * oxygen_recovery_rate

O2_makeup_kg_before_ISRU =

  gross_O2_kg

  - recovered_O2_kg

  + EVA_O2_kg

  + airlock_loss_kg

  + cabin_leakage_kg

O2_import_required_kg =

  max(0, O2_makeup_kg_before_ISRU - O2_ISRU_produced_kg)

O2_storage_required_kg =

  O2_makeup_rate_kg_per_sol

  * emergency_reserve_sols

  * storage_margin

5. Water

gross_water_kg =

  crew_count

  * water_kg_per_crew_sol

  * duration_sols

recovered_water_kg =

  gross_water_kg

  * water_recovery_rate

water_makeup_kg_before_ISRU =

  gross_water_kg

  - recovered_water_kg

  + EVA_water_kg

  + greenhouse_water_makeup_kg

water_import_required_kg =

  max(0, water_makeup_kg_before_ISRU - water_ISRU_produced_kg)

water_storage_required_kg =

  water_makeup_rate_kg_per_sol

  * emergency_reserve_sols

  * storage_margin

6. Food

gross_food_kg =

  crew_count

  * food_kg_per_crew_sol

  * duration_sols

food_produced_kg =

  gross_food_kg

  * food_closure_fraction

food_import_required_kg =

  max(0, gross_food_kg - food_produced_kg)

food_storage_required_kg =

  crew_count

  * food_kg_per_crew_sol

  * emergency_reserve_sols

  * storage_margin

7. Habitat count

habitat_count_base =

  ceil(crew_count / crew_capacity_per_habitat)

habitat_count =

  habitat_count_base + minimum_habitat_redundancy

habitat_mass_total =

  habitat_count * habitat_mass_kg

  + radiation_shielding_mass_imported_kg

  + deployment_equipment_mass_kg

8. Asset quantities

For each asset:

quantity_required =

  quantity_fixed

  + quantity_per_crew * crew_count

  + quantity_per_habitat * habitat_count

  + quantity_per_power_unit * power_unit_count

  + quantity_per_ISRU_unit * ISRU_unit_count

Round up to integer unless fractional quantities are explicitly allowed.

9. Oxygen ISRU sizing

required_O2_production_kg_per_sol =

  O2_makeup_kg_before_ISRU / duration_sols

effective_O2_output_per_unit =

  oxygen_ISRU_output_kg_per_sol

  * oxygen_ISRU_utilization_factor

  * oxygen_ISRU_availability

oxygen_ISRU_units_required =

  ceil(required_O2_production_kg_per_sol / effective_O2_output_per_unit)

oxygen_ISRU_mass =

  oxygen_ISRU_units_required * oxygen_ISRU_unit_mass_kg

oxygen_ISRU_power_kw =

  oxygen_ISRU_units_required * oxygen_ISRU_unit_power_kw

Guard against divide-by-zero if effective_O2_output_per_unit <= 0.

10. Propellant ISRU sizing

Keep this separate from survival oxygen.

propellant_O2_rate_kg_per_sol =

  propellant_O2_target_kg / propellant_production_deadline_sols

propellant_O2_ISRU_units =

  ceil(propellant_O2_rate_kg_per_sol / effective_O2_output_per_unit)

Report:

- crew_survival_ISRU_mass

- crew_survival_ISRU_power

- propellant_ISRU_mass

- propellant_ISRU_power

- total_ISRU_mass

- total_ISRU_power

11. Water extraction

water_output_kg_per_sol_per_unit =

  regolith_processed_kg_per_sol_per_unit

  * regolith_water_mass_fraction

  * water_extraction_efficiency

  * water_ISRU_availability

water_extraction_units =

  ceil(water_required_kg_per_sol / water_output_kg_per_sol_per_unit)

12. Power balance

For each asset:

asset_peak_power_kw =

  quantity * power_peak_kw

asset_continuous_power_kw =

  quantity * power_continuous_kw

asset_average_power_kw =

  quantity * power_peak_kw * duty_cycle

asset_energy_kWh_per_sol =

  asset_average_power_kw * hours_per_sol

unless explicit energy_kWh_per_sol is provided.

Total:

total_continuous_power_kw =

  sum(asset_continuous_power_kw)

  + habitat_power

  + ECLSS_power

  + ISRU_power

  + thermal_power

total_peak_power_kw =

  sum(asset_peak_power_kw) * simultaneity_factor

total_energy_kWh_per_sol =

  sum(asset_energy_kWh_per_sol)

  + habitat_energy

  + ECLSS_energy

  + ISRU_energy

  + thermal_energy

required_generation_kw =

  total_continuous_power_kw * (1 + power_margin_percent)

13. Solar sizing

solar_generation_required_kW =

  total_energy_kWh_per_sol / effective_sun_hours_per_sol

solar_generation_required_kW_derated =

  solar_generation_required_kW

  / (dust_derating_factor * seasonal_derating_factor)

solar_array_mass_kg =

  solar_generation_required_kW_derated

  / solar_array_specific_power_kw_per_kg

14. Nuclear sizing

nuclear_units_required =

  ceil(required_generation_kw / nuclear_unit_power_kw)

nuclear_mass_kg =

  nuclear_units_required * nuclear_unit_mass_kg

15. Energy storage

critical_load_kw =

  sum(loads marked critical)

autonomy_hours =

  autonomy_sols * hours_per_sol

storage_kWh_required =

  critical_load_kw

  * autonomy_hours

  / battery_roundtrip_efficiency

  / battery_depth_of_discharge

battery_mass_kg =

  storage_kWh_required / battery_specific_energy_kWh_per_kg

battery_units_required =

  ceil(battery_mass_kg / battery_unit_mass_kg)

16. Spares and sustainment

For each hardware asset:

annual_spares_kg =

  installed_asset_mass_kg * spares_percent_per_year

period_spares_kg =

  annual_spares_kg * surface_duration_earth_days / 365.25

For each resupply window:

resupply_consumables_kg =

  crew_count

  * net_consumables_kg_per_crew_sol

  * resupply_window_sols

resupply_spares_kg =

  installed_hardware_mass_kg

  * annual_spares_rate

  * resupply_window_earth_days / 365.25

resupply_total_kg =

  resupply_consumables_kg

  + resupply_spares_kg

  + replacement_hardware_kg

  + science_payload_replenishment_kg

  + medical_kg

  + contingency_kg

resupply_starships_required =

  ceil(resupply_total_kg / usable_payload_kg_per_starship)

============================================================

MISSION PACKING ALGORITHM

============================================================

Do not only compute ceil(total_mass / usable_payload). Generate dependency-aware mission manifests.

Each cargo item must have:

- asset_id

- asset_name

- mass_kg

- volume_m3

- required_phase

- earliest_mission

- latest_mission

- must_arrive_before_crew

- can_arrive_after_crew

- dependencies_prior

- dependencies_same_or_prior

- criticality: critical, important, optional

- scenario_group

Algorithm:

1. Build required asset list from scenario.

2. Expand quantities into cargo items or grouped pallets.

3. Mark critical pre-crew items:

   - power generation

   - energy storage

   - power distribution

   - communications

   - habitat/safe haven

   - ECLSS

   - thermal control

   - emergency water/O2/food stores

   - medical module

   - unloading/deployment assets

   - mobility required for unloading

4. Sort by dependency level, criticality, and phase.

5. Pack into cargo missions using:

   - usable_payload_kg_per_starship

   - maximum_volume_m3_per_starship, if available

   - dependency rules

6. Dependency rules:

   - prior dependency must be on an earlier mission.

   - same_or_prior dependency may be on the same or earlier mission.

7. Calculate mission-level cumulative values:

   - allocated_mass_kg

   - mass_margin_kg

   - allocated_volume_m3

   - volume_margin_m3

   - delivered_power_capacity_kw

   - delivered_energy_storage_kWh

   - cumulative_crew_capacity

   - cumulative_habitat_capacity

   - cumulative_life_support_capacity

   - cumulative_days_of_supply

   - dependency_status

   - readiness_status

8. Crew arrival is allowed only when all readiness checks pass.

============================================================

READINESS CHECKS

============================================================

Create red/yellow/green pass-fail checks:

Crew arrival readiness:

- habitat capacity >= crew_count

- safe-haven capacity >= crew_count

- ECLSS online

- critical power margin >= 0

- critical energy storage autonomy >= required autonomy

- oxygen storage + production supports emergency reserve

- water storage + production supports emergency reserve

- food storage supports emergency reserve

- comms online

- thermal control online

- medical/safe haven available

- unloading/deployment assets available

- minimum mobility available

- dependency rules met

- no critical asset past lifetime without spare/replacement plan

- resupply interval does not exceed stockpile endurance

Sustainment readiness:

- resupply mass/window <= cargo capacity/window

- spares replacement interval <= asset lifetime

- consumable stockpile never goes negative

- power margin remains positive during dust/autonomy case

- ISRU production does not create hidden power shortfall

- crew time demand does not exceed available crew-hours, if crew-time module is enabled
